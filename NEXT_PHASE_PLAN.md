# Next Phase Python Plan

## Goal
Move beyond the basic and advanced foundations with a balanced phase that mixes targeted revision with genuinely new practical topics.

## Recommended Study Length
- 14 more days
- 1 hour per day

## What This Phase Covers
This phase combines two needs:
- strengthen weak spots from the first 28 days
- add new topics that prepare you for real-world Python

It includes new topics you have not properly studied yet:
- JSON
- working with real data files
- custom modules in separate files
- more class methods
- class attributes vs instance attributes
- simple command-line style programs
- using external libraries
- basic API concepts

## Why This Order
You still benefit from some revision, but pure revision is not enough now.

This plan gives you both:
- stronger foundations
- new practical ground

## Main Focus Areas
- cleaner code with better variable names and structure
- stronger function design
- more work with files and dictionaries
- more class and method practice
- JSON and JSON files
- custom modules
- external libraries
- API basics

## Study Rules
- Keep lessons in tiny steps.
- Try first before seeing the full solution.
- Do one exercise at a time.
- Use projects to connect ideas.
- Revise briefly, then spend most of the session on new work.

## Daily Structure
- 5 min: Quick revision of one older pattern
- 15 min: Learn one new concept
- 30 min: Solve one exercise or mini task
- 10 min: Review mistakes and note one takeaway

## Roadmap

### Day 1: Clean Code + Better Function Design
- Better variable names
- Better spacing and formatting
- One function, one job
- Input -> process -> return

### Day 2: JSON Basics
- What JSON looks like
- Convert between Python dictionaries and JSON text
- Read simple JSON data

### Day 3: Working With JSON Files
- Save dictionary data as JSON
- Load JSON from a file
- Compare plain text files vs JSON files

### Day 4: Custom Modules
- Create your own helper file
- Import your own functions
- Split a small program into 2 files

### Day 5: Class Practice I
- Add methods to classes
- Use object data inside methods
- Create 2-3 objects from one class

### Day 6: Class Practice II
- Instance attributes vs class attributes
- Update object data
- Build a small object-based example

### Day 7: File Handling + Exception Practice
- Save reports to files
- Read data back
- Handle missing files or bad input
- Write safer programs

### Day 8: List, Set, and Dictionary Problem Solving
- Find duplicates
- Remove duplicates
- Count frequencies
- Compare readable and faster solutions

### Day 9: Simple CLI-Style Programs
- Read user input
- Show menu choices
- Run one small text-based program

### Day 10: Using External Libraries
- What a library is
- Why some imports are built-in and some are installed
- First use of `requests` or a similar beginner-friendly package

### Day 11: API Basics
- What an API is
- Request and response idea
- What JSON responses usually look like

### Day 12: Reading API Data
- Make one simple API request
- Read fields from the response dictionary
- Print selected data

### Day 13: Error Handling With APIs
- Handle failed requests
- Check missing keys safely
- Make API code more reliable

### Day 14: Mini Project + Transition Day
- Build a small project using files, JSON, functions, or APIs
- Review weak spots
- Decide the next branch:
  - APIs deeper
  - OOP deeper
  - Advanced Python features like decorators
