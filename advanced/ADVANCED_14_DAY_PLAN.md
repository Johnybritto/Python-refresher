# Advanced 14-Day Python Plan

## Goal
Build on the first 14 days and move from beginner Python into stronger problem solving, cleaner program design, and practical intermediate topics.

## Study Rules
- Keep lessons in tiny steps.
- Try first before seeing a full solution.
- Do one exercise at a time.
- Review each solution for correctness, readability, time complexity, and space complexity.
- Revisit weak spots often: string reversal, debugging, complexity, and duplicate-finding in lists.

## Daily Structure (1 Hour)
- 10 min: Read the day's notes
- 5 min: Revise one older pattern like string reversal or duplicate-finding
- 15 min: Study one new concept with examples
- 25 min: Solve one small exercise
- 5 min: Write one lesson learned

## Roadmap

### Day 1: Return Values and Function Design
- Difference between `print` and `return`
- Writing small reusable functions
- Choosing clear names and parameters

### Day 2: Scope and Variable Lifetime
- Local vs global variables
- Why function scope matters
- Avoiding accidental name conflicts

### Day 3: More With Strings
- Counting characters
- Palindrome checking
- Manual string reversal with a loop

### Day 4: More With Lists
- Building new lists from old ones
- Finding max, min, and duplicates
- Intro to list comprehensions
- Keep revising the loop pattern for finding duplicates in a list

### Day 5: Dictionaries and Nested Data
- Dictionaries of lists
- Lists of dictionaries
- Reading and updating nested values

### Day 6: Tuples and Sets in Practice
- When to use tuples
- Set operations: union, intersection, difference
- Removing duplicates in practical problems

### Day 7: Exceptions and Error Handling
- `try`, `except`
- Handling bad user input
- Writing safer programs

### Day 8: File Handling
- Reading from files
- Writing to files
- Processing simple text data

### Day 9: Modules and Imports
- Using built-in modules
- Import styles
- Splitting code into multiple files

### Day 10: Object-Oriented Programming I
- Classes and objects
- Attributes
- Methods

### Day 11: Object-Oriented Programming II
- `__init__`
- Instance behavior
- Simple class design practice

### Day 12: Testing Basics
- Why testing matters
- Writing simple `assert` statements
- Thinking in test cases

### Day 13: Intermediate Complexity and Problem Solving
- Nested loops
- Tradeoffs between readability and performance
- Comparing two solutions to the same problem

### Day 14: Intermediate Mini Project
- Combine functions, files, dictionaries, loops, and error handling
- Build one practical mini project
- Review strengths and next targets

## What Success Looks Like
By the end of these 14 days, you should be able to:
- Write functions that return useful values
- Handle common errors more safely
- Read and write simple files
- Organize code with modules
- Understand the basics of classes and objects
- Test small programs with confidence
- Compare simple solutions using time and space complexity
- Find duplicates in a list with a loop and explain how it works
